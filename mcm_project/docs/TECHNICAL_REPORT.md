# MCM 项目技术报告

## 1. 参数回收能力评估

### 1.1 回收精度实测

使用中等采集方案（15方向 × 2 b值 × 2 Δ = 60点）测试：

| 参数 | 真实值 | 无噪声误差 | SNR=100误差 | SNR=50误差 | 可辨识度 |
|------|--------|-----------|-------------|-----------|---------|
| f_ic | 0.30 | **0.013** | 0.035 | 0.058 | ✅ 高 |
| k | 5.0 | **0.4** | 0.9 | 1.9 | ✅ 中高 |
| R_ss | 4.0 μm | **1.9** | 1.9 | 1.6 | ⚠️ 低 |
| R_ls | 8.0 μm | **1.7** | 1.0 | 0.4 | ⚠️ 中低 |
| f_T | 0.80 | **0.12** | 0.12 | 0.12 | ⚠️ 中（系统性偏低）|
| μ 方向 | — | **0.1°** | 1.4° | 2.8° | ✅ 高 |
| R² | — | **1.0000** | 0.9858 | 0.9435 | — |

### 1.2 结论

**能较好回收的参数**：体积分数 f_ic、取向 μ、Watson 分散度 k。

**回收困难的参数**：球半径 R_ss/R_ls、组织水分数 f_T。

**原因分析**：
1. **R_ss/R_ls 与体积分数耦合**：球体受限扩散信号在 R=2-8μm 范围内变化平缓（见图2），小 R + 大 f 与大 R + 小 f 可以产生几乎相同的信号。这是多室模型的**已知病态问题**。
2. **f_T 系统性偏低**：拟合倾向于把部分自由水信号归因于细胞外张量（EC）隔室，因为 EC 的 d_perp 可以接近自由水扩散率。
3. **k 的敏感度依赖 b 值覆盖**：k 只在高 b 值、方向数足够多时才被充分约束。

**改善建议**：
- 增加多个扩散时间 Δ（如 15/25/40/60ms）可以更好地区分 R 和 f
- 添加低 b 值（b<500）数据点可以更好地约束 f_T
- 使用更丰富的方向采样（>30方向）改善 μ 和 k

## 2. 代码质量评估

### 2.1 当前状态

代码在功能上正确（94/94测试通过），但存在以下问题：

| 问题 | 严重程度 | 位置 | 说明 |
|------|---------|------|------|
| 导入路径依赖 sys.path | 中 | 测试文件 | 通过 `sys.path.insert` 实现，不够健壮 |
| Watson-Stick 重复生成球面网格 | **高** | `watson_stick.py` | 每次调用重新生成 60×60 网格，在拟合中累积巨大开销 |
| 单位转换分散在各处 | **高** | `mcm_forward.py` | `d_parallel_ic` 和 `d_parallel_ec` 的 1e-3 因子位置不一致 |
| mcm_signal_batch 纯 Python 循环 | 中 | `mcm_forward.py` | 60点 batch 需 8.2ms，可优化至 <2ms |
| 缺少公共 API 导出 | 低 | `src/__init__.py` | 为空文件，用户无法 `from src import ...` |
| 类型提示不完整 | 低 | 多处 | 部分函数缺少返回类型注解 |

### 2.2 是否值得重构？

**结论：值得一次中等规模重构**，重点在于：

1. **性能优化**（预估可提升 3-5 倍拟合速度）
2. **单位系统统一**（避免再出现 d_parallel 的 1e-3 bug）
3. **导入路径标准化**

### 2.3 具体重构计划

#### A. 缓存球面网格（最大性能提升）

```python
# 在 watson_stick.py 中
_GRID_CACHE = {}

def _get_grid(n_theta, n_phi):
    key = (n_theta, n_phi)
    if key not in _GRID_CACHE:
        _GRID_CACHE[key] = uniform_sphere_grid(n_theta, n_phi)
    return _GRID_CACHE[key]
```

**预期效果**：watson_stick_signal 从 0.11ms → 0.03ms（~3×）。

#### B. 批量 Watson-Stick 计算

当前 `mcm_signal_batch` 对每个方向单独调用 `watson_stick_signal`，每次都重新计算 Watson 权重 `exp(k*(μ·n)²)`。

改进：一次计算所有方向的 `q·n`，用矩阵运算替代循环。

**预期效果**：mcm_signal_batch(60点) 从 8.2ms → ~2ms。

#### C. 统一单位系统

建议：MCMParameters 中所有扩散率统一使用 **μm²/ms**，在需要时内部转换。

当前混乱状态：
- `d_parallel_ic` 在 MCMParameters 中是 1.0（μm²/ms），传给 watson_stick_signal 时需要 `*1e-3`
- `d_parallel_ec` 在 MCMParameters 中是 1.2e-3（mm²/s），传给 extracellular_tensor_signal 时也是 `*1e-3`
- `d_iso` 在 MCMParameters 中是 3.0（μm²/ms），但 free_water_signal 内部自己 `*1e-3`

这说明不同模块的单位约定不一致！

**统一方案**：
- 所有隔室模块统一接受 **mm²/s**（与文献一致）
- MCMParameters 的默认值改为 mm²/s：d_parallel_ic=1.0e-3, d_parallel_ec=1.2e-3, d_perp_ec=0.6e-3, d_iso=3.0e-3
- 移除所有 `*1e-3` 的临时转换

#### D. 可安装包结构

将 `src/` 改为 `mcm/` 并添加 `pyproject.toml`，支持 `pip install -e .`。

## 4. 实现中的关键困难

### 困难 1：球体受限扩散公式的 NaN/overflow 灾难

**症状**：直接实现 Neuman 1974、Murday-Cotts、Stepišnik 的级数公式后，小 R 值时全部溢出为 NaN，大 R 值时信号恒为 1。

**根因**：
1. **文献公式系数不一致**：Neuman 1974 的 Eq.18 系数 `1/(α_m⁶(α_m²R²-2)D)` 与 Murday-Cotts 的变体在常数因子上有差异。不同教材/论文在定义 α 时有的用有量纲（1/m）有的用无量纲（αR）。
2. **双曲函数溢出**：公式中出现 `cosh(α²DΔ/R²)`，当 R=2μm、Δ=40ms 时，指数可达 350，远超 float64 范围。
3. **本征值方程混淆**：文献中至少出现 3 种不同的球体本征值方程（j₁'(α)=0、tan(αR)=3αR/(3-(αR)²)、以及含 y j_{3/2}' 的形式），它们给出不同的根。

**解决**：放弃直接级数公式，改用**位移关联函数方法**（Grebenkov 2007 的框架）。

推导路径：
```
ln(S/S₀) = γ²G² [J₁ - J₃]
J₁ = 2∫₀^δ (δ-u) C(u) du
J₃ = ∫₀^δ dt₁ ∫_Δ^{Δ+δ} C(t₂-t₁) dt₂
C(τ) = Σ B_n [1 - exp(-λ_n τ)]   (增量关联函数)
```

将 J₁ 和 J₃ 的积分解析展开后，得到完全由指数函数组成的闭式解，彻底避免了双曲函数溢出。

**验证**：与随机游走模拟（N=20000）对比，差异 < 3%。

### 困难 2：单位转换陷阱

**症状**：随机游走模拟输出信号≈0（完全衰减），但预期应为 0.135。

**根因**：`D = 1.0 um²/ms = 1.0 × 10⁻⁹ m²/s`，但代码中写成了 `D_m = D_mm * 1e-6 = 1e-6 m²/s`，差了 1000 倍。

**影响**：步长 `std = sqrt(2Dt)` 大了 `sqrt(1000)≈31.6` 倍，导致相位累积 `γG∫x dt` 大 31.6 倍，`φ²` 大 1000 倍，信号 exp(-φ²/2) 下溢为 0。

**教训**：扩散系数的单位转换是 DWI 代码中最常见的 bug。建议统一在单位类中处理。

### 困难 3：PGSE 信号公式的符号陷阱

**症状**：关联函数方法输出信号 > 1（如 R=4μm 时 S=1.56）。

**根因**：标准布朗运动的位移关联函数有两种常见定义：
- `C(τ) = 2Dτ`（用于 SDE `dx = sqrt(2D) dW`）
- `C(τ) = Dτ`（用于 SDE `dx = sqrt(D) dW`）

PGSE 标准公式 `S = exp(-bD)` 对应 `C(τ) = Dτ`。但我们从平衡态扩散传播子推导出的 `C(τ) = 2 Σ B_n [1 - exp(-λτ)]` 对应 `C(τ) = Dτ` 的形式（因为 Σ B_n λ_n = D）。

如果错误地乘以 2，会得到 `ln(S) = -2bD`，信号 = exp(-2bD)，这比正确值小很多。

**解决**：通过自由扩散极限严格推导，确认 `C(τ)` 不应有额外的因子 2。

### 困难 4：b=0 时 Watson-Stick 数值积分不精确

**症状**：b=0 时 `watson_stick_signal` 返回 0.9983 而不是 1.0。

**根因**：60×60 的球面网格对 Watson 分布的归一化积分有 ~0.17% 的截断误差。

**解决**：添加 b=0 短路返回 1.0。同时这也是一个性能优化（跳过积分）。

### 困难 5：体积分数耦合与拟合病态

**症状**：无噪声数据中 R_ss 回收误差达 2μm，f_T 回收误差达 0.12。

**根因**：
- R_ss 与 f_ss 高度耦合：小 R + 大 f ≈ 大 R + 小 f（球体信号对 R 不敏感）
- f_T 与 EC 耦合：EC 的 d_perp 可以调整到接近自由水扩散率

**缓解**：
- 在目标函数中强制体积分数归一化
- 使用多初始值策略（multi-start）
- 测试中对 R 和 f_T 放宽容差

## 5. 性能分析与优化

### 5.1 当前性能

| 操作 | 耗时 | 占总拟合时间 |
|------|------|-------------|
| watson_stick_signal (单次) | 0.11 ms | ~55% |
| sphere_restricted_signal (单次) | 0.011 ms | ~5% |
| extracellular_tensor_signal (单次) | ~0.001 ms | <1% |
| mcm_signal_batch (60点) | 8.2 ms | — |
| fit_mcm (500 eval) | 3.5 s | — |

### 5.2 瓶颈定位

mcm_signal_batch 中：
- 对每个方向调用 watson_stick_signal（0.11ms × 60 = **6.6ms**）→ **80% 时间**
- 对每个方向调用 extracellular_tensor_signal（<0.01ms × 60）→ 可忽略
- sphere 和 free water 预计算 → 可忽略

### 5.3 优化路线图

| 优化 | 实现难度 | 预期加速 | 方法 |
|------|---------|---------|------|
| 缓存球面网格 | ⭐ 极易 | 3× | `functools.lru_cache` 或全局字典 |
| 批量 Watson-Stick | ⭐⭐ 中等 | 2-3× | 矩阵运算 `cos_qn = directions @ q_batch.T` |
| 缓存 Watson 权重 | ⭐ 极易 | 1.5× | 在 batch 内 k 不变时预计算 `exp(k*(μ·n)²)` |
| 向量化 sphere 计算 | ⭐ 极易 | 1.2× | 对相同 (b,Δ) 的点批量计算 |
| 提供解析 Jacobian | ⭐⭐⭐ 难 | 2-5× | least_squares 使用 `jac='2-point'` 较慢 |
| Numba JIT | ⭐ 极易 | 10-50× | 不可用（环境无 numba） |

**推荐立即实施**：缓存球面网格 + 批量 Watson-Stick。预估可将拟合从 3.5s → <1s。
